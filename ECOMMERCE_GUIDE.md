# オンラインストア（カート機能・決済）導入ガイド

ホームページで実際にお米を販売し、代金を決済してもらうための方法は大きく分けて3つあります。
米良氣農園のような「生産者直売」の場合、管理の手軽さとお客様への信頼感から、**パターンA（BASEやSTORES）** が最もおすすめです。

---

## パターンA：ネットショップ作成サービスを利用する（推奨）
**「BASE（ベイス）」** や **「STORES（ストアーズ）」** といった無料サービスでショップ機能を作り、ホームページの「購入ボタン」からそこへリンクさせる方法です。

*   **メリット**:
    *   **導入が簡単**: プログラムの知識不要で、すぐにカート機能が持てます。
    *   **日本の商習慣に完全対応**: クレジットカードだけでなく、コンビニ払い、銀行振込、キャリア決済、後払いなど、多様な決済方法が最初から用意されています（ここが一番大きいです）。
    *   **発送管理が楽**: 配送伝票の作成機能や、自動返信メールなどが充実しています。
*   **デメリット**:
    *   購入時だけ別サイト（BASEなどの画面）に移動することになります（違和感は少ないです）。

### 導入手順
1.  [BASE](https://thebase.in/) または [STORES](https://stores.jp/) でアカウントを作成します。
2.  商品を登録します（価格、写真、説明文など）。
3.  商品ごとに「販売ページURL」が発行されます。
4.  ホームページ（`index.html`）の「購入する」ボタンのリンク先を、そのURLに書き換えます。

```html
<!-- 例 -->
<a href="https://meraki-farm.base.shop/items/12345678" class="btn-primary">オンラインストアで見る</a>
```

---

## パターンB：埋め込み型カートを使う（Shopify Starter）
**「Shopify（ショッピファイ）」** の「スタータープラン（月額$5程度）」にある「購入ボタン（Buy Button）」機能を使います。

*   **メリット**:
    *   ホームページの中に「カートに追加」ボタンが表示され、画面遷移せずに買い物カゴが開くため、サイトのデザインを崩しません。
    *   世界的に有名なシステムなので動作が安定しています。
*   **デメリット**:
    *   月額費用がかかります。
    *   日本独自の決済（コンビニ払いなど）の設定が少し複雑な場合があります。

### 導入手順
1.  Shopifyでアカウント作成し、商品を登録。
2.  「販売チャネル」＞「Buy Button」を選択し、コードを作成。
3.  生成されたコードを `index.html` の商品部分に貼り付けます。

---

## パターンC：決済リンクを直接貼る（Stripe / Square）
**「Stripe（ストライプ）」** や **「Square（スクエア）」** の「決済リンク」機能を使います。

*   **メリット**:
    *   月額費無料で、決済手数料も安めです。
    *   もっともシンプルに「カード決済」だけを導入できます。
*   **デメリット**:
    *   「カート（まとめ買い）」機能が弱いです（商品Aと商品Bを同時に買う、などがしにくい）。
    *   基本はクレジットカード決済がメインになります。

---

## 結論：まずは パターンA がおすすめ

「親世代」や「健康志向のユーザー」がターゲットの場合、**クレジットカードを使いたくない（コンビニ払いや振込がいい）** という方も多くいらっしゃいます。
また、法律で定められた「特定商取引法に基づく表記」なども自動で生成してくれるため、**BASE** や **STORES** を開設し、ホームページからリンクを貼る形が、運営コストもリスクも低く、一番安全なスタートです。
